01 · El problema

Tiempo crítico
mal gestionado

⚠️
El proceso manual falla
En muchos centros de salud, la detección de valores críticos en analíticas depende de que un profesional revise manualmente cada resultado. Esto genera retrasos que pueden ser fatales en situaciones de emergencia: hiperpotasemia severa, glucosa crítica, troponinas elevadas.
🎯
CentineLab resuelve esto
Automatiza la detección de valores fuera de rango clínico y dispara notificaciones en tiempo real al enfermero y médico responsables de ese paciente. Trazabilidad completa desde la extracción hasta el aviso, con registro de acciones y auditoría.

02 · Circuito clínico

Del paciente
a la alerta

01
🏥
Recepción
El paciente llega con la petición que incluye códigos de barras únicos y pegatinas para los botes de analítica.
QR / Barcode
02
🩸
Extracción
El sanitario etiqueta los botes con las pegatinas del paciente. Trazabilidad garantizada, cero confusiones.
Trazabilidad
03
🔗
Asignación
El sanitario vincula la petición a un enfermero y médico mediante su ID. Cada analítica tiene responsables identificados.
ID Sanitario
04
🔔
Alerta
El sistema analiza los resultados. Si detecta valores críticos, notifica automáticamente por email y webhook.
Notificación

03 · Stack tecnológico

Construido para
escalar

FastAPI
Framework · API REST
API REST asíncrona con validación automática de esquemas mediante Pydantic. Documentación OpenAPI generada automáticamente.
SQLite / PostgreSQL
Base de datos
SQLite para desarrollo local y pruebas rápidas. PostgreSQL en producción para entornos clínicos reales con alta concurrencia.
SQLAlchemy
ORM
Modelos relacionales limpios: Paciente, Petición, Analítica, Resultado, Sanitario. Migraciones gestionadas con Alembic.
SMTP + Webhooks
Notificaciones
Sistema dual: email directo al sanitario asignado y webhook configurable para integración con Slack, Teams o sistemas hospitalarios.
Pytest
Testing
Suite de tests unitarios e integración cubriendo los rangos de alerta clínica y el flujo completo de notificación.
Docker
Despliegue
Contenedorizado para despliegue reproducible. docker-compose con servicios API, base de datos y worker de notificaciones.

04 · Motor de alertas

Valores críticos
detectados

El motor de alertas evalúa cada resultado contra rangos clínicos de referencia. La severidad determina el canal y la urgencia de la notificación. Los rangos son configurables por centro.

Parámetro Valor crítico Severidad Acción automática
Potasio (K⁺) > 6.5 mEq/L · < 2.5 mEq/L Crítica Email + Webhook inmediato
Glucosa > 500 mg/dL · < 40 mg/dL Crítica Email + Webhook inmediato
Troponina I > 0.4 ng/mL Crítica Email + Webhook inmediato
Hemoglobina < 7 g/dL · > 20 g/dL Crítica Email + Webhook inmediato
Sodio (Na⁺) > 160 mEq/L · < 120 mEq/L Alta Email en < 5 min
Creatinina > 10 mg/dL Alta Email en < 5 min
INR / Coagulación > 5.0 Alta Email en < 5 min
Leucocitos > 30.000/μL · < 2.000/μL Media Notificación en próxima revisión
Plaquetas < 50.000/μL · > 1.000.000/μL Media Notificación en próxima revisión
